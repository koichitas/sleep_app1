# 睡眠導入アプリ (Sleep Induction App) - ユーザー仕様書

## 1. アプリケーション概要

本アプリは、ユーザーが集中して数字をタップするゲームを通じて、リラックス状態を促し、最終的に自然な睡眠導入を支援することを目的とします。ゲームの進行には広告が組み込まれ、特定の条件下ではユーザーに休息を促すメッセージと広告が表示されます。

## 2. 主要機能

### 2.1. ゲームプレイ

-   **パネル形式**: N x N のグリッド形式のパネルを用意。
    -   初期ステージは 4x4 パネル (1から16の数字)。
    -   ステージクリアごとに N を増加させ、最大 10x10 パネル (1から100の数字)。
-   **数字の配置**: パネルには 1 から N*N までの数字がランダムに表示される。
-   **クリア条件**: 1から順番にパネルをタップし、すべての数字を正しくタップするとステージクリア。
-   **タップ反応**:
    -   正しくタップした場合: タップしたことが視覚的にわかるように反応を表示。
    -   間違ってタップした場合: 間違ったことが視覚的にわかるように反応を表示。
    -   どこまでタップしたかはユーザーには表示しない。ユーザーは記憶力を駆使して次の数字をタップする必要がある。
-   **連続間違い**:
    -   2回連続でタップを間違えた場合、ゲームを一時停止し、リワード広告を表示する。
    -   広告表示後、次にタップすべき正解のパネルの色を変更してヒントを与える。

### 2.2. 広告表示

-   **リワード広告**: 以下の条件でリワード広告を表示する。
    1.  **ステージクリア時**:
        -   「お疲れさまでした。次のステージに行く前に広告を見て心を整えましょう。OK」というメッセージを表示。
        -   ユーザーが「OK」をタップすると広告が表示される。
        -   広告が準備できていない場合: 30秒のカウントダウンを表示し、ユーザーに待機を促す（「広告が準備できていません。30秒後に再試行します。心を落ち着かせましょう。」など）。
    2.  **2回連続間違い時**:
        -   「2回連続で間違えました。お疲れのようですね。広告を見てゆっくりしましょう。OK」というメッセージを表示。
        -   ユーザーが「OK」をタップすると広告が表示される。
        -   広告が準備できていない場合: 上記と同様に30秒のカウントダウン。
-   **最終ステージクリア時 (10x10)**:
    -   「おめでとうございました。アプリを閉じてゆっくりやすみましょう。」というメッセージを表示。
    -   リワード広告を表示。
    -   その後、スタート画面に戻る。

### 2.3. ユーザーインターフェース (UI)

-   **スタート画面**:
    -   「睡眠導入アプリ、今日もおつかれさまでした。良い眠りを。スタート」というメッセージを表示。
    -   ゲーム開始用の「スタート」ボタンを配置。
    -   前回クリアにかかった時間、または寝落ちした時間を表示。
    -   直近3回のプレイ記録（日時、クリア時間/寝落ち時間）を表示。
-   **全体的なデザイン**:
    -   ダークモードを基調とした色使い。寝やすい、目に優しいデザインを重視。
-   **パネルサイズ**:
    -   パネルは画面の幅と高さ全体に収まるように表示される。
    -   ステージが進むにつれてパネルの数が増えるため、個々のパネルは自動的に小さくなる。

### 2.4. 寝落ち判定

-   **操作なし判定**: 5分間操作がない場合、ユーザーが「寝落ち」したと判定する。
-   **アプリ終了**: 寝落ち判定後、アプリを終了（またはスタート画面に戻る）。

### 2.5. データ記録

-   **プレイ記録**:
    -   アプリを開いた日時（何月何日何時）を記録。
    -   ステージクリアにかかった時間、または寝落ちした時間を記録。
-   **記録の表示**: スタート画面には直近3回分のプレイ記録を表示する。

### 2.6. ローカライゼーション

-   可能な限りのローカライズ対応（20言語以上）。
-   初期開発は日本語で行い、開発完了後にローカライズ対応を進める。

## 3. シナリオ

1.  ユーザーがアプリを起動。スタート画面が表示される。
2.  「スタート」ボタンをタップし、ゲーム開始。初期ステージ (4x4) が表示される。
3.  ユーザーは 1 から順番に数字をタップしていく。
4.  ステージクリアした場合、クリアメッセージと広告表示確認ダイアログが表示される。「OK」をタップすると広告が表示され、次のステージ (5x5) へ移行。
5.  途中で誤ったパネルをタップした場合、エラー反応が表示される。
6.  2回連続で間違えた場合、間違いメッセージと広告表示確認ダイアログが表示される。「OK」をタップすると広告が表示され、次にタップすべきパネルがハイライトされる。
7.  5分間操作がなかった場合、寝落ち判定となり、アプリはスタート画面に戻る（または終了）。この際、寝落ち時間が記録される。
8.  最終ステージ (10x10) をクリアした場合、最終クリアメッセージと広告表示後、スタート画面に戻る。

## 4. その他

-   **一貫性**: 他のプロジェクトで参考にできる広告実装やUI/UXのベストプラクティスは積極的に取り入れる。
